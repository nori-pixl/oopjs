仕組みの注記

oopjs.init("名前", () => { oopjs.名前 = 名前 }) — 右辺の名前はどこにも宣言されていない自由変数のはずですが、そのまま動きます。 コールバックを一度ソースコード化し、渡した文字列を実引数名として持つ本物の関数に組み直して呼び出しているためです。 with文もグローバル汚染も使いません。

fn = new Function('oopjs', '名前', 'oopjs.名前 = 名前') fn(oopjsProxy, "太郎") // "名前" が実引数として解決される
§01

部品カタログ — 10のOOP要素

オブジェクト指向の代表的な10要素を、すべてoopjs.class(...)の中だけで表現できます。 多重継承・読み取り専用定数・method_missing的フォールバックなど、素のclassには無い機能も含みます。

クラスFIG.01 / Class

データ(属性)と手続き(メソッド)を1つにまとめる設計図。実体をいくらでも量産できる。

oopjs.class("Animal", () => { oopjs.init("name", () => { oopjs.name = name }) })

カプセル化FIG.02 / Encapsulation

内部データを隠し、限定されたメソッド経由のみでアクセスさせる。不正な書き換えを防ぐ。

oopjs.private("balance", () => { oopjs.balance = 0 })

継承FIG.03 / Inheritance

既存クラスの性質を引き継いで拡張。共通コードの重複を減らす。複数指定で多重継承も可能。

oopjs.class("Duck", () => { oopjs.extends("Flyer", "Swimmer") })

ポリモーフィズムFIG.04 / Polymorphism

同じ名前のメソッドが、オブジェクトごとに違う振る舞いをする。呼び出し側のif分岐を減らせる。

shapes.map(s => s.area()) // Circle→円の面積 / Square→正方形の面積

インターフェースFIG.05 / Interface

実装を持たず、持つべきメソッドの規約だけを定義。設計と実装を分離し、結合度を下げる。

const Comparable = createInterface("Comparable", ["compareTo"]) oopjs.implements(Comparable) // 未実装は定義時にエラー

抽象クラスFIG.06 / Abstract Class

具体的な実装と、サブクラスに強制したい抽象メソッドを混在させる。共通処理を担保する。

oopjs.class("Shape", () => { oopjs.abstract("area") // 未実装ならnew時にエラー })

ダックタイピングFIG.07 / Duck Typing

継承関係に関わらず、同じメソッドを持っていれば同種として扱う。柔軟で、明示的な宣言を省ける。

oopjsDuck(obj, ["fly"]) // 継承もinstanceofも見ない、メソッドの有無だけ

トレイト / モジュールFIG.08 / Trait

多重継承の弊害を避けつつ、複数クラスでメソッド群を再利用。機能を部品化して混ぜ込む。

oopjs.trait("Loggable", () => { ... }) oopjs.use("Loggable", "Timestamped")

コンポジションFIG.09 / Composition

継承ではなく、他のオブジェクトを属性として保持して使う。関係性を柔軟にし、実行時に差し替えられる。

oopjs.has("engine", () => new oopjs.Engine()) oopjs.method("drive", () => oopjs.engine.start())

アクセス修飾子FIG.10 / Access Modifiers

private/publicなどメンバの公開範囲を制御。カプセル化を強固にし、意図しないバグを防ぐ。

無指定 = public oopjs.private(...) / oopjs.const(...) // 再代入で即エラー
§02

使い方

ビルド不要。ブラウザは<script>タグ1つ、Node.jsはESM/CommonJS両対応です。

最小の例
<script src="dist/oop-lib.js"></script> <script> oopjs.class("Person", () => { oopjs.init("name", () => { oopjs.name = name }) oopjs.method("greet", () => `こんにちは、${oopjs.name}です`) }) const p = new oopjs.Person("Nori") console.log(p.greet()) </script>

oop-lib/ ├─ src/ ESモジュール版ソース(機能別) ├─ dist/oop-lib.js ビルド不要のUMDバンドル └─ examples/ Node.js / ブラウザの動作デモ

注記

oopjs DSLは実験的機能です。Proxyと動的な関数再構築を使うため、 通常のclass構文よりデバッグはしづらくなります。 学習用途や、遊び心のあるプロトタイピング向けとして設計されています。
